# 网站样式架构

本文档概述了该网站样式文件的结构和方法。项目使用 SCSS 和 CSS，通过分层和模块化的方式来组织样式。

## 概述

样式方法可以分为三个主要类别：

1.  **基础/框架样式**：这些文件提供了全面的基础，通常来自第三方主题（如 Obsidian 主题）或项目模板。它们不应被直接编辑。
2.  **功能模块样式**：这些是小型的、单一用途的样式表，用于处理特定组件或功能的外观（例如，“复制代码”按钮、网站统计模块）。
3.  **自定义样式**：这是进行个性化和覆盖基础样式的主要层次，用于创建网站的独特外观和感觉。

所有样式文件都位于 `src/site/styles/` 目录中。

## 文件明细

### 基础/框架样式 (请勿编辑)

这些文件为网站提供了基础的外观。它们旨在成为一个稳定的基础，并且通常标记有不要直接修改的警告，因为模板更新可能会覆盖它们。

*   **`obsidian-base.scss`**: 一个全面的主题文件，模仿了 Obsidian.md 编辑器的外观。它为颜色、字体、间距以及 callouts、代码块和侧边栏等组件定义了大量的 CSS 变量。
*   **`1.css`**: 一个被压缩过的 CSS 文件，其内容是 "Minimal Theme by @kepano"。它作为另一个基础层，提供了一套极简的样式和变量。
*   **`digital-garden-base.scss`**: 此文件包含专为“数字花园”模板设计的样式。它处理核心组件（如知识图谱、文件树导航、侧边栏和搜索功能）的布局和外观。
*   **`style.scss`**: 一个更通用的基础样式表，可能来自最初的模板。它为颜色设置了根变量，并为标题、链接和引用块定义了非常基本的样式。

### 自定义样式 (主要编辑文件)

所有特定于项目的修改和覆盖都应放在这里。

*   **`custom-style.scss`**: 这是网站定制中最重要的文件。它覆盖了基础样式，以实现网站的独特设计。此处定义的关键功能包括：
    *   **主题化**：为亮色 (`.theme-light`) 和暗色 (`.theme-dark`) 主题定义颜色。
    *   **响应式设计**：广泛使用 `@media` 查询来调整导航、侧边栏和内容在不同屏幕尺寸下的布局。
    *   **组件样式**：为文件树、目录、卡片等提供自定义外观。
    *   **交互效果**：定义了光标点击效果 (`.cursor-ripple`, `.cursor-particle`) 的样式。

### 功能模块/实用工具样式

这些文件是模块化的，用于处理离散功能的样式。

*   **`copy-code.css`**: 为代码块上的“复制”按钮提供样式。
*   **`lazy-loading.css`**: 为图片懒加载提供样式，包括占位符的闪烁效果和防止内容布局偏移（CLS）的技术。
*   **`mobile-optimization.css`**: 包含用于改善移动设备可用性的特定规则，例如确保触摸目标（按钮、链接）足够大以便于点击（WCAG 标准）。
*   **`site-stats.css`**: 为显示网站统计信息（如页面计数、字数等）的部分提供卡片式布局的样式。

## 样式架构

样式表按特定顺序加载，以创建级联效果：

1.  **基础层**：首先加载像 `obsidian-base.scss`, `digital-garden-base.scss` 和 `style.scss` 这样的文件，以建立默认外观。
2.  **模块层**：加载功能特定的文件，如 `copy-code.css` 和 `site-stats.css`。
3.  **定制层**：最后加载 `custom-style.scss`。由于它最后加载，其规则具有更高的优先级，可以轻松覆盖基础样式并定义网站的最终外观。

这种分层方法是一种最佳实践，因为它将定制与核心主题文件分离开来，使项目更易于维护和更新。
